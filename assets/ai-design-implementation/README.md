# AI Design & Implementation Page Graphics

Recommended graphic assets for the AI Design & Implementation methodology page (`/agentic-ai.html`).

This folder is reserved for visual assets that support that page. Drop final files here using the suggested filenames so HTML references can be wired without renaming.

## Hero Image

Suggested filename:
`ai-design-implementation-hero.png`

Purpose:
Full-width hero background image for the AI Design & Implementation methodology page.

Visual direction:
Structured, abstract operating-system architecture. Deep navy and FAA blue gradients. Dimensional layers, controlled light, no readable text, no UI overlays, no typography.

## Section Image: Misapplied AI

Suggested filename:
`misapplied-ai-operating-risk.png`

Purpose:
Optional section visual for the "Most AI Isn't Failing. It's Being Misapplied." section.

Visual direction:
A business workflow under stress: fragmented handoffs, duplicated records, disconnected decision paths, and controlled tension. Premium consulting-grade editorial style. No readable text.

## Section Image: Human-in-the-Loop Control

Suggested filename:
`human-in-the-loop-control-layer.png`

Purpose:
Optional section visual for FAA's approach to control, review, escalation, and accountability.

Visual direction:
A structured decision environment with humans reviewing exceptions inside a layered workflow system. Professional, architectural, operational, no text.

## Usage Notes

- Do not place text inside images.
- Images should support the page copy, not carry the message.
- Keep FAA palette: deep navy, FAA blue gradients, white highlights, subtle dimensional depth.
- Images should feel like Bain / BCG / EY-Parthenon insight imagery, not generic AI art.

## Current Wiring

The page currently uses `assets/global-page-hero-background-flow.png` as its hero background (preserved from the prior version). Once `ai-design-implementation-hero.png` is added here, the hero `background-image` reference in `agentic-ai.html` can be updated.

The two optional section images are not currently referenced in markup. Wiring them in is optional and should be coordinated with the surrounding copy so the visuals support — not duplicate — the message.
